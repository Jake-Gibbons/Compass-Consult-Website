## Overview

The site is deployed to **Netlify** via automatic Git integration. Every
push to the `main` branch triggers a new production deployment. Build
settings, cache headers, security headers, and serverless function
configuration are all defined in [`netlify.toml`](../netlify.toml) at
the repository root.

**Live site:** https://compassconsult.co.uk

## Primary Deployment — Netlify

### How it works

1. Push changes to the `main` branch on GitHub.
2. Netlify detects the push via its GitHub integration.
3. Netlify runs the build command defined in `netlify.toml`:
   ```toml
   [build]
     publish = "."
     command = "npm run build:css"
   ```
4. The built site (entire repository root) is published.
5. Netlify Functions in `netlify/functions/` are bundled with esbuild
   and deployed as serverless functions automatically.

The Lucide Icons bundle and versioned assets are pre-built and committed
to the repository, so only Tailwind CSS needs to be compiled during the
Netlify build.

### First-time setup

1. Go to [netlify.com](https://netlify.com) and log in.
2. Click **Add new site → Import an existing project**.
3. Connect your GitHub account and select this repository.
4. Netlify will auto-detect the `netlify.toml` settings — no manual
   configuration is required.
5. Click **Deploy site**.

### Custom domain

1. Go to **Site Settings → Domain management**.
2. Click **Add a domain** and enter `compassconsult.co.uk`.
3. Update the DNS records at your registrar (see [DNS Configuration](#dns-configuration)).
4. Netlify provisions a free SSL certificate via Let's Encrypt automatically.

### Netlify Forms

Contact and newsletter forms use Netlify's built-in form handling.
Forms must include the `netlify` HTML attribute:

```html
<form name="contact" method="POST" netlify>
```

Submissions are accessible in the Netlify dashboard under **Forms**.
Configure email notifications in **Site Settings → Forms → Form notifications**.

## CI/CD — GitHub Actions

Three workflows run automatically on push to `main` or on pull requests:

| Workflow | File | Trigger | Purpose |
|---|---|---|---|
| Lint & Link Check | `.github/workflows/check.yml` | Push / PR to `main` | Runs `npm run check` on Node 18 and 20 |
| Generate Sitemap & Robots | `.github/workflows/main.yml` | Push to `main` | Regenerates `sitemap.xml` and `robots.txt` and commits the result |
| Sync Resources | `.github/workflows/sync-resources.yml` | Push to `main` | Synchronises downloadable resource files |

### Lint & Link Check workflow

```yaml
# .github/workflows/check.yml
on:
  push:
    branches: [main]
  pull_request:
    branches: [main]
jobs:
  check:
    strategy:
      matrix:
        node-version: [18.x, 20.x]
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-node@v4
        with:
          node-version: ${{ matrix.node-version }}
          cache: 'npm'
      - run: npm ci
      - run: npm run check
```

### Sitemap & Robots workflow

Runs after every push to `main`, generates `sitemap.xml` from all HTML
files using their last-commit date as `<lastmod>`, and commits the
updated files back to the repository.

## Alternative Hosting Options

While Netlify is the primary host, the site is a standard static site
and can be hosted anywhere.

### Vercel

1. Push code to GitHub.
2. Go to [vercel.com](https://vercel.com) → **New Project**.
3. Import the GitHub repository.
4. Set the build command to `npm run build:css` and the output directory to `.`.
5. Add environment variables (see [Environment Variables](#environment-variables)).
6. Deploy.

> **Note:** Netlify Forms and Netlify Functions will not work on Vercel.
> You will need alternative form and serverless function providers.

### GitHub Pages

1. Go to repository **Settings → Pages**.
2. Set the source branch to `main` and the folder to `/ (root)`.
3. Save — the site deploys to `https://jake-gibbons.github.io/Compass-Consult-Website/`.

> **Note:** Netlify Forms and Netlify Functions are not available on
> GitHub Pages. The contact form will not submit without an alternative backend.

### Traditional Hosting (cPanel / FTP)

1. Run `npm run build` locally to ensure assets are compiled and versioned.
2. Connect via FTP/SFTP using credentials from your host.
3. Upload all files (excluding `.git/`, `node_modules/`) to `public_html/`.
4. Verify the site loads in a browser.

**Recommended FTP clients:**
- [FileZilla](https://filezilla-project.org/) (free, cross-platform)
- [Cyberduck](https://cyberduck.io/) (free, macOS / Windows)
- [WinSCP](https://winscp.net/) (free, Windows)

## Environment Variables

The following environment variables are required for the Netlify Functions
to work in production. Set them in the Netlify dashboard under
**Site Settings → Environment Variables**.

| Variable | Description |
|---|---|
| `NETLIFY_BLOBS_*` | Auto-provisioned by Netlify — no manual setup required |

No other environment variables are needed for the static site itself.

## DNS Configuration

Point `compassconsult.co.uk` to Netlify:

```
Type     Name    Value
A        @       75.2.60.5          ← Netlify load balancer IP
CNAME    www     [site-name].netlify.app
```

Or use Netlify DNS (recommended — enables automatic SSL and branch deploys):

1. In the Netlify dashboard, go to **Domains → Add or register domain**.
2. Follow the instructions to delegate your domain to Netlify's nameservers.

## SSL / HTTPS

Netlify provisions and renews SSL certificates automatically via
Let's Encrypt for all custom domains. No manual configuration is required.

To enforce HTTPS on other Apache-based hosts, add to `.htaccess`:

```apache
RewriteEngine On
RewriteCond %{HTTPS} off
RewriteRule ^(.*)$ https://%{HTTP_HOST}%{REQUEST_URI} [L,R=301]
```

## Post-Deployment Tasks

After deploying, verify the following:

1. **Verify the live site**
   - Visit https://compassconsult.co.uk and all page URLs.
   - Test the contact form end-to-end.
   - Verify all images load.

2. **Search Engine Optimisation**
   - Submit `https://compassconsult.co.uk/sitemap.xml` to
     [Google Search Console](https://search.google.com/search-console).
   - Submit to [Bing Webmaster Tools](https://www.bing.com/webmasters).
   - Monitor indexing status for any errors.

3. **Security**
   - Confirm HTTPS is enforced (no mixed-content warnings).
   - Check security headers are applied using
     [securityheaders.com](https://securityheaders.com).

4. **Performance**
   - Run [PageSpeed Insights](https://pagespeed.web.dev/) on the homepage.
   - Review [Core Web Vitals](https://web.dev/vitals/) scores.
   - Check the Lighthouse report against the committed
     `lighthouse-production-report.json` baseline.

5. **Monitoring**
   - Confirm uptime monitoring is active.
   - Check Netlify function logs for any errors.

## Performance Optimisation

### Images

```bash
# Optimise new images before deploying
npm run optimize:images
```

Or use online tools: [Squoosh](https://squoosh.app/),
[TinyPNG](https://tinypng.com/), [ImageOptim](https://imageoptim.com/).

### Caching

Cache headers are configured in `netlify.toml`:

| Asset type | Cache-Control |
|---|---|
| HTML pages | `max-age=0, must-revalidate` |
| CSS / JS (versioned) | `max-age=31536000, immutable` |
| Images | `max-age=31536000, immutable` |
| Downloads | `max-age=604800` (1 week) |

## Rollback Procedure

### Option 1 — Netlify dashboard (fastest)

1. Go to the Netlify dashboard → **Deploys**.
2. Find the last known-good deployment.
3. Click **Publish deploy** to instantly restore it.

### Option 2 — Git revert

```bash
# Revert the problematic commit
git revert <commit-hash>
git push origin main
# Netlify automatically deploys the reverted commit
```

### Option 3 — Git reset (force push, use with caution)

```bash
git reset --hard <good-commit-hash>
git push --force origin main
```

After rolling back:

1. Identify the root cause.
2. Fix in a feature branch.
3. Test thoroughly locally.
4. Re-deploy via a new Pull Request to `main`.

## Troubleshooting Deployment

### Site not loading

- [ ] Check DNS is pointing to the correct Netlify IP / CNAME.
- [ ] Verify the deploy succeeded in the Netlify dashboard.
- [ ] Check for build errors in the Netlify deploy log.

### Slow performance

- [ ] Run `npm run optimize:images` and redeploy.
- [ ] Confirm versioned assets are being served with immutable cache headers.
- [ ] Check Netlify CDN is serving assets from a nearby edge node.

### Forms not working

- [ ] Confirm the form has the `netlify` attribute.
- [ ] Verify form submissions appear in the Netlify dashboard under **Forms**.
- [ ] Check spam filters — legitimate submissions may be flagged.
- [ ] Test with a different email address.

### Functions not working

- [ ] Check function logs in the Netlify dashboard under **Functions**.
- [ ] Confirm environment variables are set correctly.
- [ ] Test the function endpoint directly.

### SSL certificate issues

- [ ] Check certificate status in **Site Settings → Domain management**.
- [ ] Force a certificate renewal in the Netlify dashboard.
- [ ] Clear the browser cache and retry.

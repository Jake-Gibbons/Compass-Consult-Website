## рџЊЌ Overview

Compass Consult (Employability & Skills) Ltd works with individuals,
employers, and public sector authorities to deliver tailored employability
and skills programmes across the UK. This repository contains the full
source code for the company's public-facing website.

The site is a **fully responsive static website** built without any
front-end framework. It is structured for clarity, performance, and
scalability, and is deployed on Netlify with serverless functions for
form handling and subscriber management.

### вњЁ Key Features

- **Fully responsive** вЂ” mobile-first layout with a dedicated sidebar
  navigation on desktop and a slide-in hamburger menu on mobile
- **Progressive Web App (PWA)** вЂ” installable on mobile and desktop via
  a Service Worker (`sw.js`) and web app manifest
- **Accessibility-first** вЂ” skip navigation links, semantic HTML,
  ARIA attributes, keyboard-navigable menus, and `prefers-reduced-motion`
  support throughout
- **Performance optimised** вЂ” locally built and versioned CSS/JS assets
  with long-lived cache headers; WebP images with lazy loading
- **SEO ready** вЂ” XML sitemap (auto-generated by CI), `robots.txt`,
  Open Graph meta tags, and structured Schema.org markup
- **Security hardened** вЂ” CSP, `X-Frame-Options`, `X-Content-Type-Options`,
  and other security headers enforced via `netlify.toml`
- **Netlify Forms** вЂ” contact and newsletter forms with zero-config
  spam filtering
- **Netlify Functions** вЂ” serverless TypeScript function for subscriber
  management backed by Netlify Blobs storage
- **Automated CI/CD** вЂ” GitHub Actions for linting, link checking, and
  sitemap/robots.txt regeneration on every push to `main`

- [Node.js](https://nodejs.org/) 16 or higher
- A modern code editor ([VS Code](https://code.visualstudio.com/) recommended)
- Git

### Run locally

```bash
# 1. Clone the repository
git clone https://github.com/Jake-Gibbons/Compass-Consult-Website.git
cd Compass-Consult-Website

# 2. Install dependencies
npm install

# 3. (Optional) Install Git hooks
npm run hooks:install

# 4. Start the local dev server
npm run serve
# в†’ http://localhost:8000

# 5. (Optional) Watch and rebuild Tailwind CSS on change
npm run watch:css
```

## рџЏ— Architecture Notes

### Navigation

Every page uses the same two-piece navigation pattern:

| Element | Visible at |
|---|---|
| `<aside>` вЂ” fixed left sidebar | Desktop (в‰Ґ 1024 px) |
| `<nav>` вЂ” top bar + hamburger menu | Mobile (< 1024 px) |

The hamburger menu is wired up in `js/main.js` в†’ `initializeMobileMenu()`.

### JavaScript (`js/main.js`)

`main.js` is the single shared script loaded by every page. It runs all
initialisers on `DOMContentLoaded`. Each function is documented with JSDoc
comments. Key functions:

| Function | Purpose |
|---|---|
| `initializeMobileMenu()` | Hamburger menu toggle with ARIA support |
| `initializeStickyTalkButton()` | Floating "Talk to Us" CTA button |
| `initializeRevealAnimations()` | IntersectionObserver scroll reveals |
| `initializeBioReadMore()` | Expandable team bio sections |
| `enhanceExternalLinks()` | `noopener`/`noreferrer` + aria-label injection |
| `connectFormLabels()` | Programmatically links `<label>` to `<input>` |
| `optimizeImages()` | Adds lazy-loading and async decoding attributes |
| `initializeAOS()` | Starts the Animate On Scroll library |

A global `window.CompassConsult` object exposes `scrollToElement`,
`debounce`, `isInViewport`, and the two animation initialisers for use
by individual page scripts.

### Styling (`css/main.css`)

`main.css` supplements Tailwind with:

- CSS custom properties for the brand colour palette
- Hamburger icon animation (three bars в†’ Г—)
- Mobile menu slide-in transition
- Sticky "Talk to Us" button and tooltip styles
- Scroll-triggered reveal animation utilities
- Reduced-motion media query overrides

Tailwind CSS is **built locally** (not loaded from CDN) using the CLI:
`npm run build:css` compiles `css/input.css` в†’ `css/tailwind.min.css`,
and `npm run build:assets` then appends a content hash for cache busting.

See the table of contents at the top of `main.css` for a full section list.

### Data Files (`data/`)

JSON files store team member details, service descriptions, and client
information as a single source of truth for the site's content. When
updating content, edit both the JSON file and the corresponding HTML page
to keep them in sync. See [`data/README.md`](data/README.md) for the
full field schema for each file.

### Netlify Functions (`netlify/functions/`)

`subscribers.mts` is a TypeScript serverless function that handles
newsletter subscription requests. It persists subscriber records to
Netlify Blobs and is bundled automatically by Netlify's esbuild bundler
on deployment.

### Asset Versioning

`scripts/version-assets.mjs` reads each asset file, computes a truncated
SHA-256 content hash, and renames the file (e.g.
`tailwind.min.css` в†’ `tailwind.min.a1b2c3d4.css`). An `assets-manifest.json`
file maps original names to versioned names so HTML files can reference
the correct filename. This enables long-lived `Cache-Control: immutable`
headers without stale-cache issues.

### Progressive Web App

`sw.js` is a Service Worker that pre-caches the homepage shell on install
and serves cached assets on subsequent visits. The PWA manifest in
`assets/icons/favicon/` declares the app name, icons, theme colour, and
display mode so the site can be added to a device's home screen.

## рџљЂ Deployment

The site is deployed to **Netlify** on every push to `main`.
Build settings and HTTP headers are defined in [`netlify.toml`](netlify.toml).

The Netlify build command is `npm run build:css`. Lucide Icons and asset
versioning are pre-built and committed, so the Netlify build step only
needs to recompile Tailwind CSS.

GitHub Actions run two additional automation workflows on push to `main`:

| Workflow | File | Purpose |
|---|---|---|
| Lint & Link Check | `check.yml` | ESLint + broken-link audit (Node 18 & 20) |
| Sitemap & Robots | `main.yml` | Auto-generate `sitemap.xml` and `robots.txt` |

### Other hosting options

| Platform | Notes |
|---|---|
| **Netlify** | Primary host вЂ” built-in form handling and serverless functions |
| **Vercel** | Zero-config static hosting with global CDN |
| **GitHub Pages** | Free; Settings в†’ Pages в†’ select `main` branch |
| **FTP / cPanel** | Upload all files to `public_html` via FileZilla or Cyberduck |

See [`docs/DEPLOYMENT.md`](docs/DEPLOYMENT.md) for detailed deployment
instructions including environment variables, DNS configuration, and
rollback procedures.

## рџ”’ Security

Security headers are enforced globally via `netlify.toml`:

- `Content-Security-Policy` вЂ” restricts resource origins
- `X-Frame-Options: DENY` вЂ” prevents clickjacking
- `X-Content-Type-Options: nosniff` вЂ” prevents MIME sniffing
- `Referrer-Policy: strict-origin-when-cross-origin`
- `Permissions-Policy` вЂ” disables camera, microphone, and geolocation

# Project Structure

This document is a detailed reference for every folder and file in the
Compass Consult Website repository.

---

## Root Directory

```
Compass-Consult-Website/
├── index.html                        # Main homepage (entry point)
├── sw.js                             # Service Worker — PWA offline support
├── sitemap.xml                       # XML sitemap (auto-generated by CI)
├── robots.txt                        # Crawl directives (auto-generated by CI)
├── assets-manifest.json              # Maps original asset names to versioned names
├── tailwind.config.js                # Tailwind CSS configuration and brand tokens
├── eslint.config.mjs                 # ESLint flat config for JS linting
├── netlify.toml                      # Netlify build settings, headers, and functions
├── package.json                      # npm scripts, dependencies, and project metadata
├── package-lock.json                 # Dependency lock file
├── README.md                         # Project overview and quick-start guide
├── CONTRIBUTING.md                   # Contribution guidelines
├── STRUCTURE.md                      # This file — detailed folder/file reference
└── _gen_thumbs.py                    # Python helper script for thumbnail generation
```

## `pages/`

Secondary HTML pages, all sharing the same `<head>`, skip link, sidebar
`<aside>`, mobile `<nav>`, and `<footer>` as the homepage.

| File | Page |
|---|---|
| `about.html` | About Compass Consult |
| `company-information.html` | Company registration, legal details, and structure |
| `contact.html` | Contact form and office contact information |
| `cookies.html` | Cookie policy |
| `events.html` | News, events, and announcements |
| `partnerships.html` | Partnerships, accreditations, and affiliations |
| `privacy.html` | Privacy policy (GDPR-compliant) |
| `resources.html` | Downloadable resource library (factsheets, guides) |
| `services.html` | Full listing of services offered |
| `sitemap.html` | HTML sitemap for human visitors |
| `team.html` | Team member profiles and biographies |
| `terms.html` | Terms of service |
| `work-for-us.html` | Careers page and current vacancies |

---

## `css/`

| File | Description |
|---|---|
| `input.css` | Tailwind CSS entry file — `@tailwind` directives and custom layer declarations |
| `main.css` | Custom styles: brand CSS variables, hamburger animation, mobile menu transitions, sticky CTA button, scroll-reveal utilities, and reduced-motion overrides |
| `tailwind.min.[hash].css` | Compiled and minified Tailwind output with a content hash for cache busting |
| `aos.css` | AOS (Animate On Scroll) library stylesheet |

---

## `js/`

| File | Description |
|---|---|
| `main.js` | Single shared script loaded on every page. Initialises mobile menu, sticky button, scroll animations, bio read-more, external link enhancement, form labels, image lazy-loading, and AOS. Exposes utilities via `window.CompassConsult`. |
| `lucide.min.[hash].js` | Lucide Icons bundle (built by `scripts/build-lucide.mjs` and versioned) |
| `aos.min.[hash].js` | AOS library bundle (versioned) |

## `assets/`

### `assets/images/`

Site imagery, organised into sub-folders:

| Folder | Contents |
|---|---|
| `authorities/` | Logos and images for client organisations / local authorities |
| `events/` | Photos and graphics for the news & events page |
| `partners/` | Partner organisation logos |
| `resource-previews/` | Thumbnail previews for downloadable resources |
| `team/` | Profile photos for team members |
| `work-for-us/` | Imagery for the careers / vacancies page |

### `assets/logos/`

Brand logos in multiple formats for use across the site and external materials:

| File | Description |
|---|---|
| `Logo.png` | Full-colour Compass Consult logo (PNG) |
| `Logo.webp` | Full-colour Compass Consult logo (WebP, optimised) |
| `compass_rose.png` | Compass rose icon (PNG) |
| `compass_rose.svg` | Compass rose icon (SVG, scalable) |
| `optimized/` | Pre-optimised versions of the above for use in HTML |

### `assets/icons/favicon/`

PWA-ready favicon set including:

- `favicon.ico` — legacy browser favicon
- `favicon-[size].png` — PNG icons at various sizes
- `apple-touch-icon.png` — iOS home screen icon
- `site.webmanifest` — Web App Manifest (name, icons, theme colour, display mode)

### `assets/downloads/`

Downloadable PDF documents served from the resources page:

| Folder | Contents |
|---|---|
| `docs/` | General information factsheets and guides |
| `vacancies/` | Job description and vacancy PDFs |

## `data/`

JSON files that act as a single source of truth for the site's structured
content. Editing these files alone is not sufficient — the corresponding
HTML pages must also be updated to stay in sync.

| File | Description |
|---|---|
| `README.md` | Full field schema reference for all JSON files |
| `clients.json` | Client / authority objects shown in the homepage ticker strip |
| `services.json` | Service definitions (name, description, features, pricing) |
| `team.json` | Team member profiles (name, role, bio, photo, contact details) |

See [`data/README.md`](data/README.md) for the complete field-by-field schema.

## `netlify/`

Netlify platform integration files.

| Path | Description |
|---|---|
| `functions/subscribers.mts` | TypeScript serverless function that handles newsletter subscription requests and persists subscriber data to Netlify Blobs |
| `lib/data.ts` | Shared data-access utilities used by the serverless functions |

---

## `scripts/`

Node.js scripts invoked by `npm run` commands. See `package.json` for the
full list of aliases.

| File | npm script | Description |
|---|---|---|
| `build-lucide.mjs` | `build:lucide` | Bundles Lucide icon set into a single minified file using esbuild |
| `generate-pwa-icons.mjs` | `generate:pwa-icons` | Generates the full PWA icon set from the source logo using sharp |
| `link-check.mjs` | `link-check` | Crawls all HTML files and reports broken internal and external links |
| `optimize-images.mjs` | `optimize:images` | Batch-compresses images to WebP / optimised JPEG using sharp |
| `sync-resources.mjs` | `sync:resources` | Synchronises downloadable resource files |
| `version-assets.mjs` | `build:assets` | Appends a content hash to CSS/JS filenames and writes `assets-manifest.json` |
| `perf-update.mjs` | *(manual)* | Performance audit and update helper |

## `.github/`

GitHub-specific configuration files.

### `.github/workflows/`

| File | Trigger | Purpose |
|---|---|---|
| `check.yml` | Push / PR to `main` | Runs `npm run check` (ESLint + link check) on Node 18 and 20 |
| `main.yml` | Push to `main` | Auto-generates `sitemap.xml` and `robots.txt` and commits the result |
| `sync-resources.yml` | Push to `main` | Syncs downloadable resource files |

---

## `.githooks/`

Git hooks registered via `npm run hooks:install` (sets `core.hooksPath`
to `.githooks/`). These run automatically before commits or pushes to
enforce quality gates locally.

---

## Configuration Files

| File | Description |
|---|---|
| `tailwind.config.js` | Tailwind CSS content paths, brand colour tokens, custom animations (`float`, `blob`, `infinite-scroll`), typography plugin, and safelist |
| `eslint.config.mjs` | ESLint flat config targeting `js/**/*.js` |
| `netlify.toml` | Netlify publish directory (`.`), build command, functions directory, cache headers per asset type, and security headers |
| `package.json` | Project metadata, npm scripts, dev/production dependencies, Node engine constraint (`≥ 16`), and `browserslist` |
